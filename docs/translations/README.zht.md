## 為什麼選擇 OpenTaint？

> OpenTaint 是 *Semgrep Pro* 和 *CodeQL* 的開源替代方案——一個可自訂、可自行託管的形式化過程間污點分析引擎，專為讓 AI 代理驅動你的安全分析而打造，無需在每次掃描時消耗 token。

AI 產生生產程式碼的速度已經超過了安全團隊能夠跟上的節奏，而為捕捉其錯誤而生的兩類工具，各自都帶來了糟糕的取捨：

- **AST 模式匹配器**（Semgrep OSS、ast-grep、各類 linter）免費且快速，但它們匹配的是語法，而非資料流——跨越函式邊界或持久層的不可信輸入會直接溜過去。真正能捕捉這類問題的、更深入的過程間分析，長期以來一直被鎖在專有工具之中。
- **LLM 安全代理**能發現模式匹配器遺漏的問題，但每次執行都要重新讀取你的程式碼。token 會隨著每個檔案、每次提交、每次 CI 建置不斷累積——而機率模型仍然無法保證它捕捉了一切。

OpenTaint 以靜態分析器的成本，為你提供 LLM 代理的深度：

- **發現 AST 模式匹配器遺漏的問題。** 形式化的過程間資料流引擎能夠跨函式邊界、持久層、別名和非同步程式碼追蹤不可信資料。
- **只為模型付費一次，而非每次掃描。** 讓代理將單個發現提煉為一條污點規則。確定性引擎隨後會在整個程式碼庫——以及之後的每一次提交——中重播該規則，僅需數分鐘的 CPU 時間，且零 token 成本。
- **開源，開箱即用。** 引擎、規則和 CI 整合作為一個整體技術堆疊，基於 Apache 2.0 和 MIT 授權發布。

## 快速開始

**安裝指令碼 (Linux/macOS)**
```
curl -fsSL https://opentaint.org/install.sh | bash
```

**透過 Homebrew 安裝 (Linux/macOS)：**
```bash
brew install --cask seqra/tap/opentaint
```

**安裝指令碼 (Windows PowerShell)**
```
irm https://opentaint.org/install.ps1 | iex
```

**透過 npm 安裝 (Linux/macOS/Windows)：**
```bash
npm install -g @seqra/opentaint
```

**或使用 npx 立即執行——無需安裝（需要 Node.js）：**
```bash
npx @seqra/opentaint scan
```

**掃描你的專案：**
```bash
opentaint scan
```

**或使用 Docker：**
```bash
docker run --rm -v $(pwd):/project -v $(pwd):/output \
  ghcr.io/seqra/opentaint:latest \
  opentaint scan --output /output/results.sarif /project
```

更多選項請參閱[安裝指南](../../docs/README.md#installation)和[使用方法](../../docs/README.md#usage)。

## AI 代理工作流程

OpenTaint 內含代理技能，可將靜態分析轉化為端對端的應用安全工作流程。透過以下指令安裝：

```bash
npx skills add https://github.com/seqra/opentaint
```

`appsec-agent` 技能可協調完整的專案評估：建置專案、執行 OpenTaint、探索攻擊面、新增針對性規則、建模缺失的函式庫資料流、對發現結果進行分級，並可選擇為已確認的漏洞產生動態概念驗證檢查。

內含的技能涵蓋常見的安全分析循環：

- **掃描與分級：** `build-project`、`run-scan`、`analyze-findings`、`generate-poc`
- **覆蓋範圍擴展：** `triage-dependencies`、`discover-attack-surface`、`create-test-project`、`create-rule`、`assemble-lib-rules`
- **資料流建模：** `analyze-external-methods`、`create-pass-through-approximation`、`create-dataflow-approximation`、`debug-rule`、`report-analyzer-issue`
